Items

Items

  • To find all the items in the item master and to create new items navigate to the main module and select 'Items'

Add Items

  • To add / create new items click on + New icon provided in the top right corner of the Items page.

Primary Details:

  • Select Entity and Item Type in the primary details section. Need to select from the item type from the standard list as shown below.

  • Select the Item Type to differentiate the items.
  • The Item Type Asset-IT, Asset-Movable, Asset-Immovable are related to assets of the organizations.
  • The Item Type Consumable are items that are intended to be consumed.
  • The Item Type Service are Work Order/Service related items.
  • The Item Type Trading/Finished Goods are items for sales.
  • The Item Type Trading Serial No are related to any serial or batch items. These items can have an expiry date as well.
  • The Item Type Raw Material are related to raw materials normally used in production.
  • The Item Type Trading/Semi Finished Goods are related to semi finished goods.
  • The Item Type Scrap Item are related to scrap Items. If scrap type is selected, please enter the Scrap Amount under Finance Related Details section.

 

Item Details:

  • Enter the Item No. The Item No. can also be configured as an automated system generated sequential number.
  • Enter Description of the item by providing all the information with specifications.
  • Enter Additional Description.
  • UOM (Unit of Measure) is defined as the standard unit of measurements used when accounting for stock, and expressing them in quantities. In UOM enter the primary (Base) UOM of the item.

  If an Item is managed with multiple UOMs, uncheck Alternate UOM.

  • When Alternate UOM is unchecked, a Sub UOM field will be opened. The sub UOM can be used to represent the denominations of the primary (Base) UOM. By clicking on Add UOM multiple sub UOM lines can be added for that particular Item.

1. First line in the Sub UOM will show the actual (Primary) UOM entered.

2. Click on Add UOM to create new rows for multiple sub UOMs. Enter the sub UOMs and also fill the Qty value (This is nothing but the conversion factor/rate of the sub UOM with respect to the primary UOM). Based on the value entered in Qty, system will calculate and convert the sub UOM quantity into to the primary UOM quantity to store it in the inventory.

3. There is an option to check the default UOM. If the default box is checked, then the associated sub UOM will be appeared as a default UOM while doing transactions for that item.

4. Sub UOM can be removed by clicking on (x) under Action Column.

  Enter the Manufacturer and Manufacturer Part No of the Item.

  Click on the Category and select the category in which the item belongs to.

  Click on the Image and browse to attach the item image.

  Click on the Attachment and browse to attach any document relevant to the item.

 

 

Purchase Related Details:

  Enter Minimum Order Qty (The minimum quantity that must be purchased in one order) of that particular Item.

  Enter Order Multiples (The order multiple is an integer that defines the quantity to which an order quantity is rounded up).

 

Inventory Related Details:

  Select the Criticality of the item - High, Medium or Low.

  • Enter the Item Barcode.

  Enter the Item Expiry Days if necessary.

  • If the item has to go through an inspection process before in warding into inventory, then check Inspection Required.
  • Select the Issue Method from the drop down Manual or Backflush.
  • If issue method is selected as Manual, then at the time of issuing an item from the warehouse, the item details from the stock like batch number has to be selected manually.
  • If the Method is selected as Backflush, then select the Issue Method Type as FIFO- First In First Out or LIFO- Last In First Out. Based on the selected type, at the time of issuing the item from the warehouse, the item details from the stock like batch number will be automatically assigned by the system.

  • Catch Weight Item: Used to capture the actual weight of an item. It is a parallel unit of measure used to manage variable weight products. If the item is defined as a catch weight item, then the system will provide an option to capture its actual weight when in-warding and the same will be stored in inventory/warehouse. The catch weight stored in the inventory will get updated automatically in the whenever a transaction happens. Catch Weight is applicable for Batch or Serial items only.
  • If the Catch Weight Item is checked, then enter the nominal weight (Pre defined actual weight of the item) and select its weight UOM.
  • Invoice By Weight: If checked, then at the time of invoicing the item price will be calculated based on the Catch Weight of the item.

 

Finance Related Details:

  • Enter HSN/SAC code of the Item as per government standards. (HSN Stands for Harmonized System Nomenclature and SAC stands for Service Accounting Code) This is basically used to classify the item (Goods are classified under HSN and Services under SAC) and to define the taxation slabs based on the codes.
  • Select Cess Type percentage if applicable for the item. Cess is the tax levied by the government for any specific purpose.
  • Enter Cess Percentage and Cess Amount if Cess Types is selected.

  Select Purchase Account of that particular Item. This is the purchase ledger associated to that item.

  Select Sales Account of that particular Item. This is the sales ledger associated to that item.

  Select Finance Group of that Particular Item. This is the finance group ledger associated to that item.

  • Enter Scrap Amount if the Item Type is Scrap Item. This is the price value of the scrap item for accounting purpose.

 

Depreciation Details:

  • Enter Depreciation details if the item is an asset.
  • Select Depreciation Method from the drop down.
  • Select Depreciation Based On from the drop down.
  • Enter Useful Life(Years) or Useful Life(Months) of the asset.
  • Enter the Salvage Value of the Asset (The estimated resale value of an asset at the end of its useful life).
  • Enter Depreciation Percentage.
  • Enter Depreciation Frequency.
  • Select Depreciation Expense Account (Dr) from the ledger.
  • Select Accumulated Depreciation Account (Cr) from the ledger.

 

Scrap Item:

Once the item becomes scrap, to account it as a scrap item, an associated Scrap Item has to be created in the system.

  • To link the item to its scrap item, select the already created Scrap Item of the item, by clicking on Scrap Item Search and click Add.

  • After adding all the details of the items, click on Create.

 

  All the items that are created so far can be found under the Items tab.

  • Bulk Upload is used to upload all the Items and details at once.
  • Click on Arrow Mark icon in Items page and Select Import Items.

  • To download bulk upload template Click on Sample xls. In the downloaded excel template the cells with (*) are mandatory fields which needs to be filled before uploading.

Material Requests - Pending

Materials Ready for Dispatch

  • Based on the material request, if the material is ready to dispatch (to inform the requester that the material is ready in the ware house for delivery), select that line item and click on the three dots icon provided under action column and then click on Materials Ready for Dispatch.

 

  • Click on Submit.
  • Once done, the selected item will disappear from Material Request - Pending sub tab and will be moved to Materials Ready for Dispatch sub tab for further action.

 

Deliver Materials

  • Based on the material request, to deliver the requested materials, click on the three dots icon provided under action column and click on Deliver Materials.
  • To select all the line items of a particular material request, click on the Material Request No. highlighted in blue colour.

  • Select Billing Address from the drop down list.
  • Select Delivery Address from the drop down list.
  • Enter Vehicle No.
  • Enter Payment Terms.
  • Check Alternate Delivery Address and enter the details, if the delivery address is a new one, which is not available in the drop down list.
  • Enter the delivering quantity in Qty field. System will allow partial delivery as well.
  • If any quantity is already delivered, it will be displayed in the Delivered Qty field.

 

  • Click on Submit.
  • Once done, the selected item will disappear from Material Request - Pending sub tab and will be moved to Materials Delivered sub tab under Material Deliveries for further action.
  • In case of partial delivery, the request will be still available in the Material Request Pending sub tab, with status as Partially Delivered until the requested quantity of that line item get delivered completely.

 

Cancel Request

  • Select appropriate Status from the drop down list.

  • Click on Submit.
  • Once done, the selected item will disappear from Material Request - Pending sub tab and the selected cancellation status will be updated in the Material Request tab.

Suppliers

  • To create new suppliers, click on Suppliers under the main Module.

 

Create Suppliers

 

  • Click on New in the Suppliers page.

 

Primary Details:

  • Select Entity.
  • Enter Supplier Name.
  • Enter Supplier ID. Supplier ID can also be configured as an automated system generated sequential number.
  • Link to Customer can be used if the supplier has to be configured as a customer as well. Check the Link to Customer box and start typing the customer name and select the customer from the dropdown, then all the details from the customer master will be added to the creation page. Edit or add the details required and click on Create Supplier.

 

  • When same supplier is in different locations and need to enter both the supplier name, use Supplier Name Alias option. Alias Name is a common name for a particular supplier having multiple locations. For ex: supplier 1- ABC Corporation Ghana, supplier 2 ABC Nigeria. If alias name is given as ABC INC for both of the suppliers, then in the invoice the supplier name will be printed as ABC INC as a common name. If Supplier Name Alias is used then that name will be printed on all the PDFs.
  • Select the pre-defined Market Segment (Like Demographic, Geographic etc) from the dropdown.

 

Company Details:

  • Select Registration Type. Registration type is to identify the type of taxation the customer falls into.
  • Select Sector/Enterprise Type.
  • Select the predefined Line of Business.
  • Select the Legal Structure of the company.
  • If RCM (Reverse charging mechanism) is applicable check RCM checkbox, GSTIN is not required in this case.
  • Enter other details as well wherever applicable.

 

Address Details:

  • Enter the address details Information of the Supplier.

 

Contact Details:

  • Enter the contact details of the supplier.

 

Additional Contact Details:

  • Additional contact details of the supplier can be added by clicking on Add Another Contact option.
  • To remove Additional Contact Details, click on X icon.

 

Payment Related Details:

  • Accounts: Select the pre-defined accounting ledger type to which the supplier has to be linked.
  • Enter the payment details of the suppliers. The Payment Terms and Payment Days mentioned will be displayed in the purchase order.

 

Bank Details/Cheque No:

  • To enter the bank details of the supplier, select the Bank name from the list and enter the details. Click on Add another Bank to add details of multiple banks. To remove the added bank information click on the X icon.

 

Summary:

  • Enter any comments in the Comments section.
  • Attachments can be used to attach any related documents by clicking on Browse File.
  • Multiple attachments can be done by clicking on Add another File.

  • After entering all the required details of the supplier, click on Create Supplier.

Purchase Orders

Purchase Requisition

 

  • To create a new purchase requisition, click on Purchase Orders under the main module

  • Click on New in the Purchase Requisitions page.

 

Create Purchase Requisition

 

Primary Details:

  • Select respective Entity from the drop down.
  • Select supplier from the Supplier drop down.
  • Select currency from Currency list.
  • Select Approval Flow from the drop down. The approval workflow is a pre-configured process flow used to set the level of approvals and associated approvers in each level. Based on the selected workflow, system will sent the created requisition to the approvers. Once it is approved the requisition will be converted to a Purchase Order.
  • If Email PO to Supplier is checked then once the purchase order (PO) is created an e-mail along with the PO will be sent automatically to the supplier.
